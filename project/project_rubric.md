# TCSS 445/545 Database System Design
## Final Project: B+ Tree Index

### Overview
In this assignment you will extend the simple database system you built in previous projects by implementing a B+ tree index. The index will allow efficient point-wise (e.g., equality) and range queries on a single column. You will implement the core B+ tree data structure (insertion and search) and integrate it into your system’s query execution flow so that appropriate `SELECT` statements automatically use the index instead of scanning the entire table.

### System Design
Your index will be implemented as an in‑memory B+ tree (persistence is not required). The tree maintains key‑value pairs where the key is the indexed column’s value and the value is a record identifier (e.g., student ID, employee ID) that allows your system to quickly locate the full record.  
Leaf nodes are linked to support efficient range queries. You must decide on the degree of the tree (which determines the maximum number of keys per node) and handle node splits during insertion.

### Implementation
* The program should not use external database libraries or applications.  
* Acceptable programming languages: Python, Java, C/C++, C#, Rust.  
* The B+ tree must be built from scratch; you may not use existing tree/collection libraries that already provide B+ tree functionality.

### Functionalities
* **SQL extension:** Support the `CREATE INDEX index_name ON table_name (column_name);` statement to build an index on an existing table.  
* **Data loading:** Support the `LOAD DATA INFILE 'filename.csv' INTO TABLE tablename;` statement that reads a CSV file and inserts rows into the specified table.  
* **Query acceleration:** When executing a `SELECT ... FROM t WHERE column = value`, your system must use the index if one exists on that column, bypassing a full table scan.  
* **Range query acceleration:** Support `SELECT ... FROM t WHERE column > a AND column < b` (and similar) via index range scan utilizing the linked leaf nodes if an index exists on the column.
